WHAT IS IT?
PyTorch is a deep learning framework (library) for building and training neural 
networks. It provides:
- Tensor operations (like NumPy but on GPU)
- Automatic differentiation (backpropagation)
- Neural network modules and layers
- Optimization algorithms
- GPU acceleration

Website: pytorch.org
Developed by: Meta AI Research

WHY USE IT?
- Industry standard for deep learning research
- Easy to debug (eager execution by default)
- Flexible architecture design
- Excellent GPU support
- Large community and documentation
- Integrates well with other libraries

CORE COMPONENTS:

torch.nn (Neural Network Module):
  - Container for neural network layers
  - Base class for building models
  
  Example:
    import torch.nn as nn
    
    class MyModel(nn.Module):
        def __init__(self):
            super().__init__()
            self.fc = nn.Linear(10, 5)
        
        def forward(self, x):
            return self.fc(x)

torch.optim (Optimization):
  - SGD: Stochastic Gradient Descent with momentum
  - Adam: Adaptive learning rate optimizer
  - Others: AdamW, RMSprop, etc.
  
  Example:
    optimizer = torch.optim.SGD(
        model.parameters(),
        lr=0.001,
        momentum=0.937
    )

torch.Tensor:
  - Multi-dimensional array
  - Fundamental data structure
  - Supports GPU acceleration
  
  Example:
    x = torch.tensor([1, 2, 3])  # CPU
    y = torch.cuda.FloatTensor([1, 2, 3])  # GPU

torch.cuda:
  - GPU acceleration interface
  - Check GPU availability
  - Move data/models to GPU
  
  Example:
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model = model.to(device)
    x = x.to(device)

WHERE USED IN PROJECT?
- Kaggle training notebook: Model definition, training loop
- Local inference: Model loading, inference execution
- Backbone architecture: Tensor operations

KEY FUNCTIONS IN PROJECT:

torch.manual_seed(42):
  Sets random seed for reproducibility
  Why: Ensures same results across runs
  Usage: At the start of training/inference

torch.cuda.is_available():
  Checks if GPU is available
  Why: Decide device (cuda vs cpu)
  Usage: In initialization

model.train() / model.eval():
  Sets model to training or evaluation mode
  Why: Different behavior for dropout, batch norm
  Usage: Before training loop and validation

torch.no_grad():
  Disables gradient calculation
  Why: Faster inference, saves memory
  Usage: Inside validation/inference loops

torch.nn.utils.clip_grad_norm_():
  Clips gradient norm to prevent explosion
  Why: Stabilize training with large gradients
  Usage: After loss.backward()

────────────────────────────────────────────────────────────────────────────
3.1 Basic Neural Network Layers
────────────────────────────────────────────────────────────────────────────

nn.Conv2d (Convolutional Layer):
  
  WHAT:
  Applies convolutional filters to image
  Learns feature patterns
  
  Parameters:
  - in_channels: Number of input channels (3 for RGB)
  - out_channels: Number of output channels (number of filters)
  - kernel_size: Size of convolutional kernel (e.g., 3 for 3x3)
  - stride: Step size of convolution
  - padding: Zero-padding to preserve size
  - dilation: Spacing between kernel elements (dilated conv)
  
  Example:
    conv = nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1)
    # Takes 3-channel image, outputs 64 feature maps
    # Using 3x3 kernel with 1-pixel padding
  
  Output shape:
    Input: (batch, channels, height, width)
    Output: (batch, out_channels, height, width)
  
  WHERE USED:
  - Backbone feature extraction
  - Modified architecture with dilated convolutions

nn.BatchNorm2d (Batch Normalization):
  
  WHAT:
  Normalizes feature activations within each batch
  Stabilizes training by reducing internal covariate shift
  
  Parameters:
  - num_features: Number of channels
  - eps: Small value to prevent division by zero
  - momentum: Momentum for running statistics
  
  Example:
    bn = nn.BatchNorm2d(64)
    x = conv(x)
    x = bn(x)
  
  Benefits:
  - Faster training
  - Less sensitive to initialization
  - Acts as regularization (reduces need for dropout)
  
  WHERE USED:
  - After each convolution in backbone
  - Standard practice in modern networks

nn.ReLU, nn.SiLU (Activation Functions):
  
  WHAT:
  Introduces non-linearity into network
  Without activation, layers are just linear transformations
  
  ReLU (Rectified Linear Unit):
    f(x) = max(0, x)
    Simple, fast, effective
  
  SiLU (Sigmoid Linear Unit):
    f(x) = x * sigmoid(x)
    Smoother than ReLU, used in modern architectures
    YOLO v8 uses SiLU
  
  Example:
    activation = nn.ReLU(inplace=True)
    x = activation(x)
    
    OR modern:
    x = nn.SiLU()(x)
  
  WHY NEEDED:
  - Enable learning non-linear relationships
  - Without activation: f(f(f(x))) = Ax + b (still linear!)
  
  WHERE USED:
  - After convolution and batch norm
  - Core part of every modern network layer

nn.Linear (Fully Connected Layer):
  
  WHAT:
  Applies linear transformation: y = xW^T + b
  
  Parameters:
  - in_features: Input dimension
  - out_features: Output dimension
  
  Example:
    fc = nn.Linear(256, 3)  # 256 inputs -> 3 outputs (for 3 classes)
  
  WHERE USED:
  - Not used much in detection (uses convolutional heads)
  - Used in attention mechanisms for channel recalibration

nn.AdaptiveAvgPool2d (Global Average Pooling):
  
  WHAT:
  Reduces spatial dimensions to 1x1
  Averages over all spatial locations per channel
  
  Example:
    pool = nn.AdaptiveAvgPool2d(1)
    # Input: (batch, channels, height, width)
    # Output: (batch, channels, 1, 1)
  
  WHERE USED:
  - Attention mechanisms (squeeze spatial dimension)
  - Converting spatial features to channel features

nn.Sequential (Model Container):
  
  WHAT:
  Container for ordered sequence of layers
  
  Example:
    model = nn.Sequential(
        nn.Conv2d(3, 64, 3, 1, 1),
        nn.BatchNorm2d(64),
        nn.ReLU(inplace=True),
        nn.Conv2d(64, 128, 3, 1, 1),
        nn.BatchNorm2d(128),
        nn.ReLU(inplace=True)
    )
    
    output = model(input_tensor)
  
  Alternative: Use nn.Module and define forward() explicitly

3.2 Advanced Components Used in Modified YOLO
────────────────────────────────────────────────────────────────────────────

DILATED CONVOLUTION (Atrous Convolution):
  
  WHAT:
  Convolution with gaps between kernel elements
  Increases receptive field without increasing parameters
  
  Parameters:
  - dilation=1: Standard convolution (no gaps)
  - dilation=2: Kernel spread over 5x5 area (for 3x3 kernel)
  - dilation=4: Kernel spread over 7x7 area
  
  Receptive Field Calculation:
    RF = kernel_size + (kernel_size - 1) * (dilation - 1)
  
  Example:
    # Standard 3x3: RF = 3
    conv1 = nn.Conv2d(64, 128, 3, dilation=1)
    
    # Dilated 3x3 with d=2: RF = 5
    conv2 = nn.Conv2d(64, 128, 3, dilation=2, padding=2)
    
    # Dilated 3x3 with d=4: RF = 7
    conv3 = nn.Conv2d(64, 128, 3, dilation=4, padding=4)
  
  Visual:
    Standard Conv (d=1):     Dilated Conv (d=2):
    X X X                    X . X
    X X X                    . X .
    X X X                    X . X
    
  WHERE USED IN PROJECT:
  - Enhanced backbone in modified architecture
  - Improved receptive field for context understanding

RESIDUAL CONNECTIONS (Skip Connections):
  
  WHAT:
  Direct connection from input to output of a layer/block
  Instead of: y = f(x)
  Now: y = f(x) + x (element-wise addition)
  
  Benefits:
  - Gradient flow in very deep networks
  - Addresses vanishing gradient problem
  - Enables training of networks with 100+ layers
  - Empirically improves performance
  
  Example:
    class ResidualBlock(nn.Module):
        def __init__(self, channels):
            super().__init__()
            self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
            self.bn1 = nn.BatchNorm2d(channels)
            self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
            self.bn2 = nn.BatchNorm2d(channels)
        
        def forward(self, x):
            residual = x
            out = self.conv1(x)
            out = self.bn1(out)
            out = nn.ReLU()(out)
            out = self.conv2(out)
            out = self.bn2(out)
            out = out + residual  # Skip connection!
            return out
  
  WHERE USED IN PROJECT:
  - Enhanced FPN with residual connections
  - Improved feature fusion

ATTENTION MECHANISMS:

Channel Attention (SE-Block):
  
  Process:
  1. Global Average Pool: (B,C,H,W) -> (B,C,1,1)
  2. FC layers with bottleneck (reduce to C/16, then back to C)
  3. Sigmoid: Output weights between 0 and 1
  4. Multiply with original features
  
  Purpose:
  - Recalibrate channels
  - Focus on disease-relevant feature channels
  - Suppress irrelevant channels
  
  Example:
    class ChannelAttention(nn.Module):
        def __init__(self, channels, reduction=16):
            super().__init__()
            self.avg_pool = nn.AdaptiveAvgPool2d(1)
            self.fc = nn.Sequential(
                nn.Linear(channels, channels // reduction),
                nn.ReLU(),
                nn.Linear(channels // reduction, channels),
                nn.Sigmoid()
            )
        
        def forward(self, x):
            b, c, _, _ = x.size()
            y = self.avg_pool(x).view(b, c)
            y = self.fc(y).view(b, c, 1, 1)
            return x * y

Spatial Attention:
  
  Process:
  1. Channel-wise max and avg pooling
  2. Concatenate to 2 channels
  3. Conv 7x7 layer
  4. Sigmoid: Output spatial weights
  5. Multiply with original features
  
  Purpose:
  - Focus on disease-affected regions
  - Suppress irrelevant spatial areas
  - Learn which image regions are important
  
  Example:
    class SpatialAttention(nn.Module):
        def __init__(self, kernel_size=7):
            super().__init__()
            self.conv = nn.Conv2d(2, 1, kernel_size, 
                                  padding=kernel_size//2)
            self.sigmoid = nn.Sigmoid()
        
        def forward(self, x):
            avg = x.mean(dim=1, keepdim=True)
            max_val, _ = x.max(dim=1, keepdim=True)
            x_cat = torch.cat([avg, max_val], dim=1)
            return x * self.sigmoid(self.conv(x_cat))

FEATURE PYRAMID NETWORK (FPN):
  
  WHAT:
  Multi-scale feature representation
  Combines features from different scales
  
  Process:
  1. Backbone extracts features at multiple scales
  2. Top-down pathway: Upsample coarse features
  3. Lateral connections: Merge coarse and fine features
  4. Output: Multi-scale feature maps
  
  Purpose:
  - Detect objects at different scales
  - Small objects: Use fine-grained features
  - Large objects: Use coarse features
  - Efficient computation
  
  Example (Simplified):
    class SimpleFPN(nn.Module):
        def forward(self, features):
            # features = [f_small, f_medium, f_large]
            
            # Upsample large to medium scale
            large_up = upsample(features[2])
            
            # Add to medium features
            medium = features[1] + large_up
            
            # Upsample to small scale
            medium_up = upsample(medium)
            small = features[0] + medium_up
            
            return [small, medium, features[2]]
  
  WHERE USED IN PROJECT:
  - Enhanced FPN in modified architecture
  - Better multi-scale feature extraction

DECOUPLED DETECTION HEADS:
  
  WHAT:
  Separate prediction branches for classification and localization
  
  Instead of:
    features -> conv -> [class_logits, bbox_coords]
  
  Use:
    features -> shared_conv
             -> class_branch -> class_logits
             -> bbox_branch -> bbox_coords
  
  Advantages:
  - Task-specific feature learning
  - Reduced gradient conflicts
  - Better optimization for each task
  
  Example:
    class DecoupledHead(nn.Module):
        def __init__(self, in_channels, num_classes):
            super().__init__()
            
            # Shared backbone
            self.shared = nn.Sequential(
                nn.Conv2d(in_channels, in_channels, 3, padding=1),
                nn.ReLU()
            )
            
            # Classification branch
            self.cls = nn.Sequential(
                nn.Conv2d(in_channels, in_channels, 3, padding=1),
                nn.Conv2d(in_channels, num_classes, 1)
            )
            
            # Regression branch
            self.reg = nn.Sequential(
                nn.Conv2d(in_channels, in_channels, 3, padding=1),
                nn.Conv2d(in_channels, 4, 1)  # 4 bbox coords
            )
        
        def forward(self, x):
            shared = self.shared(x)
            class_pred = self.cls(shared)
            reg_pred = self.reg(shared)
            return class_pred, reg_pred

  - Modified architecture detection head
  - Better separation of tasks

────────────────────────────────────────────────────────────────────────────
4.1 Loss Functions
────────────────────────────────────────────────────────────────────────────

BINARY CROSS-ENTROPY (BCE):
  
  WHAT:
  Loss for binary classification problems (yes/no, disease/no-disease)
  
  Formula:
    BCE = -[y*log(p) + (1-y)*log(1-p)]
    
    y: Ground truth (0 or 1)
    p: Predicted probability [0, 1]
  
  Interpretation:
  - When y=1: Loss = -log(p), penalizes if prediction is < 1
  - When y=0: Loss = -log(1-p), penalizes if prediction is > 0
  
  PyTorch:
    loss_fn = nn.BCELoss()
    loss = loss_fn(predictions, targets)
    
    OR with logits (numerical stability):
    loss_fn = nn.BCEWithLogitsLoss()
  
  WHERE USED IN PROJECT:
  - Classification loss (per class)
  - Objectness loss (is there an object?)
  - Attention sigmoid outputs

INTERSECTION OVER UNION (IoU) BASED LOSS:
  
  IoU Definition:
    IoU = |A ∩ B| / |A ∪ B|  (intersection / union)
    Ranges from 0 to 1
  
  IoU Loss (Simple):
    L_IoU = 1 - IoU
    Minimizing this loss maximizes IoU
  
  CIoU (Complete IoU) - USED IN PROJECT:
    Also penalizes:
    - Center point distance
    - Aspect ratio mismatch
    
    Better gradient flow than simple IoU
    Better convergence
  
  PyTorch (using external library or custom):
    # Custom implementation
    def ciou_loss(boxes1, boxes2):
        # Calculate IoU
        inter = calculate_intersection(boxes1, boxes2)
        union = calculate_union(boxes1, boxes2)
        iou = inter / union
        
        # Calculate center distance
        c = center_distance_squared(boxes1, boxes2)
        d = diagonal_distance_squared(boxes1, boxes2)
        
        # Calculate aspect ratio difference
        ar = aspect_ratio_penalty(boxes1, boxes2)
        
        # Combine
        ciou = iou - (c/d) - ar
        loss = 1 - ciou
        return loss
  
  WHERE USED IN PROJECT:
  - Bounding box regression loss
  - Main component of detection loss

4.2 Optimization Techniques
────────────────────────────────────────────────────────────────────────────

SGD WITH MOMENTUM:
  
  WHAT:
  Stochastic Gradient Descent with momentum
  Accelerates convergence in relevant directions
  
  Update Rule (simplified):
    v = momentum * v + gradient  # Velocity accumulation
    weights -= lr * v
  
  Momentum (momentum=0.937 in YOLO):
  - Accumulates gradient direction over steps
  - Smooths noisy updates
  - Helps escape shallow local minima
  - High momentum (0.9+) = more acceleration
  
  PyTorch:
    optimizer = torch.optim.SGD(
        model.parameters(),
        lr=0.001,
        momentum=0.937,
        weight_decay=0.0005
    )
  
  WHERE USED IN PROJECT:
  - Standard optimizer for YOLO
  - Training loop in Kaggle notebook

LEARNING RATE SCHEDULING:
  
  WHAT:
  Changes learning rate during training
  
  Strategies:
  
  Constant:
    lr = 0.001 (entire training)
    Simple but suboptimal
  
  Step Decay:
    Decrease lr by factor every N epochs
    lr = lr * 0.1 at epoch 30, 60, etc.
  
  Cosine Annealing (USED IN PROJECT):
    lr = lr_min + 0.5*(lr_max - lr_min)*(1 + cos(π*epoch/epochs))
    
    Smoothly decreases from max to min
    Allows "jumping out" of minima late in training
    Benefits:
    - Smooth convergence
    - Avoids sharp learning rate drops
    - Standard practice in modern training
  
  Warm-up (USED IN PROJECT):
    First 3-5 epochs: Gradually increase lr from 0
    Reason: Stabilize initial training
    Prevents wild gradient swings with bad initial weights
  
  PyTorch:
    scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
        optimizer,
        T_max=50  # Number of epochs
    )
    
    for epoch in range(epochs):
        train(...)
        scheduler.step()  # Update lr

WEIGHT DECAY (L2 REGULARIZATION):
  
  WHAT:
  Adds penalty term to loss for large weights
  
  Loss = MainLoss + lambda * ||weights||²
  
  Effect:
  - Prevents weights from growing too large
  - Reduces overfitting
  - Weight decay=0.0005 is standard
  
  PyTorch:
    optimizer = torch.optim.SGD(
        params,
        lr=0.001,
        weight_decay=0.0005  # L2 regularization strength
    )

GRADIENT CLIPPING:
  
  WHAT:
  Limits maximum gradient norm
  
  Problem Addressed:
  - Gradients can explode during training
  - Causes numerical instability
  - Loss becomes NaN
  
  Solution:
  - If ||gradient|| > max_norm, scale down to max_norm
  - Preserves direction, limits magnitude
  
  PyTorch:
    loss.backward()
    torch.nn.utils.clip_grad_norm_(
        model.parameters(),
        max_norm=10.0
    )
    optimizer.step()

IoU CALCULATION:

def calculate_iou(box1, box2):
    """
    Calculate Intersection over Union
    
    box1, box2: [x1, y1, x2, y2] format
    """
    # Calculate intersection area
    x_left = max(box1[0], box2[0])
    y_top = max(box1[1], box2[1])
    x_right = min(box1[2], box2[2])
    y_bottom = min(box1[3], box2[3])
    
    if x_right < x_left or y_bottom < y_top:
        return 0.0
    
    intersection = (x_right - x_left) * (y_bottom - y_top)
    
    # Calculate union area
    box1_area = (box1[2] - box1[0]) * (box1[3] - box1[1])
    box2_area = (box2[2] - box2[0]) * (box2[3] - box2[1])
    union = box1_area + box2_area - intersection
    
    iou = intersection / union
    return iou

PRECISION AND RECALL:

def calculate_precision_recall(predictions, ground_truth, iou_threshold=0.5):
    """
    predictions: List of detected boxes with confidence
    ground_truth: List of true boxes
    """
    tp = 0
    fp = 0
    fn = 0
    
    for pred_box, conf in predictions:
        matched = False
        for gt_box in ground_truth:
            iou = calculate_iou(pred_box, gt_box)
            if iou >= iou_threshold:
                tp += 1
                matched = True
                break
        
        if not matched:
            fp += 1
    
    fn = len(ground_truth) - tp
    
    precision = tp / (tp + fp) if (tp + fp) > 0 else 0
    recall = tp / (tp + fn) if (tp + fn) > 0 else 0
    
    return precision, recall

NON-MAXIMUM SUPPRESSION (NMS):

def nms(boxes, scores, iou_threshold=0.45):
    """
    Remove overlapping detections, keep highest confidence
    
    boxes: List of [x1, y1, x2, y2]
    scores: Confidence for each box
    iou_threshold: Remove if IoU > threshold
    """
    # Sort by score (descending)
    order = np.argsort(scores)[::-1]
    
    keep = []
    while len(order) > 0:
        i = order[0]
        keep.append(i)
        
        if len(order) == 1:
            break
        
        # Calculate IoU with remaining boxes
        ious = np.array([
            calculate_iou(boxes[i], boxes[j])
            for j in order[1:]
        ])
        
        # Keep only boxes with IoU < threshold
        order = order[1:][ious < iou_threshold]
    
    return keep

MEAN AVERAGE PRECISION (mAP):

def calculate_ap(precisions, recalls):
    """
    Calculate Area Under Precision-Recall Curve
    """
    # Ensure starting from (0, 0)
    precisions = [0] + precisions + [0]
    recalls = [0] + recalls + [1]
    
    # Calculate area under curve
    ap = 0
    for i in range(len(recalls) - 1):
        # Use maximum precision for monotonic interpolation
        p = max(precisions[i:])
        ap += p * (recalls[i+1] - recalls[i])
    
    return ap

def letterbox(img, new_shape=(640, 640), color=(114, 114, 114)):
    """
    Resize image preserving aspect ratio with padding
    
    Purpose:
    - Maintain original proportions
    - No distortion
    - No information loss
    - Standard YOLO preprocessing
    """
    h, w = img.shape[:2]
    r = min(new_shape[0]/h, new_shape[1]/w)  # Scale ratio
    
    # New unpadded size
    new_h, new_w = int(h*r), int(w*r)
    
    # Resize
    img = cv2.resize(img, (new_w, new_h))
    
    # Add padding
    dh = (new_shape[0] - new_h) // 2
    dw = (new_shape[1] - new_w) // 2
    
    img = cv2.copyMakeBorder(
        img, dh, new_shape[0]-new_h-dh,
        dw, new_shape[1]-new_w-dw,
        cv2.BORDER_CONSTANT, value=color
    )
    
    return img

YOLO FORMAT CONVERSION:

def bbox_xyxy_to_xywh(bbox):
    """
    Convert [x1,y1,x2,y2] to [cx,cy,w,h] (YOLO format)
    """
    x1, y1, x2, y2 = bbox
    cx = (x1 + x2) / 2
    cy = (y1 + y2) / 2
    w = x2 - x1
    h = y2 - y1
    return [cx, cy, w, h]

def bbox_xywh_to_xyxy(bbox):
    """
    Convert [cx,cy,w,h] to [x1,y1,x2,y2]
    """
    cx, cy, w, h = bbox
    x1 = cx - w/2
    y1 = cy - h/2
    x2 = cx + w/2
    y2 = cy + h/2
    return [x1, y1, x2, y2]

YOLO ANNOTATION PARSING:

def load_yolo_annotation(txt_file, img_width, img_height):
    """
    Load YOLO format annotation
    Returns boxes in pixel coordinates
    """
    boxes = []
    with open(txt_file, 'r') as f:
        for line in f:
            parts = line.strip().split()
            class_id = int(parts[0])
            
            # Normalized coordinates
            cx, cy, w, h = [float(x) for x in parts[1:]]
            
            # Convert to pixel coordinates
            x1 = int((cx - w/2) * img_width)
            y1 = int((cy - h/2) * img_height)
            x2 = int((cx + w/2) * img_width)
            y2 = int((cy + h/2) * img_height)
            
            boxes.append({
                'class': class_id,
                'bbox': [x1, y1, x2, y2]
            })
    
    return boxes

def inference_single_image(model, image_path, conf_threshold=0.25):
    """
    Complete inference pipeline for single image
    """

    image = cv2.imread(image_path)
    original_shape = image.shape

    input_tensor = preprocess_image(image)

    # 3. Forward pass
    with torch.no_grad():
        predictions = model(input_tensor)
    
    # 4. Parse predictions
    detections = []
    for pred in predictions:
        if pred['conf'] > conf_threshold:
            # Convert normalized coords to pixel coords

            detections.append({
                'class': class_names[pred['class']],
                'confidence': pred['conf'],
                'bbox': bbox
            })
    
    # 5. Apply NMS
    detections = nms(detections, iou_threshold=0.45)
    
    return detections

DEBUGGING COMMON ISSUES:

1. Out of Memory (OOM)
   Problem: CUDA out of memory error
   Solution:
   - Reduce batch size (16 → 8)
   - Reduce image size (640 → 512)
   - Use smaller model (yolov8m → yolov8s)
   - Clear GPU cache: torch.cuda.empty_cache()

2. NaN Loss
   Problem: Loss becomes NaN during training
   Solution:
   - Reduce learning rate
   - Use gradient clipping (already done)
   - Check data for invalid values
   - Use mixed precision training

3. Poor Accuracy
   Problem: Model not learning
   Solution:
   - Increase training epochs
   - Check data augmentation
   - Verify data loading (labels correct)
   - Try higher learning rate
   - Use learning rate scheduler

4. Slow Training
   Problem: Training takes very long
   Solution:
   - Reduce image size
   - Use smaller batch (faster iteration)
   - Use mixed precision
   - Use distributed training (multiple GPUs)

PERFORMANCE OPTIMIZATION:

1. Faster Inference
   - Quantization (8-bit weights)

   - Batch processing (process multiple images together)
   - Remove unnecessary preprocessing

2. Lower Memory Usage
   - Gradient accumulation
   - Mixed precision (FP16)

   - Larger model

   - Ensemble of models
